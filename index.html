1.1. Kaleidoscope语言

本教程使用一种名为“万花筒”（Kaleidoscope）的玩具语言进行说明（源于含义“美丽、形式和视野”）。万花筒是一种过程式语言，允许你定义函数、使用条件语句、数学运算等。在本教程中，我们将扩展万花筒以支持if/then/else构造、for循环、自定义运算符、JIT编译和简单的命令行界面、调试信息等。

我们希望保持简单，所以万花筒中唯一的数据类型是64位浮点类型（在C语言中称为“double”）。因此，所有值都是隐式双精度的，并且该语言不需要类型声明。这使该语言具有非常漂亮和简单的语法。例如，下面计算斐波那契数列的简单示例：

使用LLVM构建我的第一个语言前端.00 - 介绍

发表于 2025-05-18 分类于 计算机技术 ， 编译器

要求: 该教程假设你了解C++，但不需要你有过编译器相关的经验。

欢迎来到“我的第一个语言前端”教程。我们将通过实现一个简单的语言来展示实现编译器的乐趣和简单性。本教程将帮助你快速入门，并展示一个使用LLVM生成代码的具体示例。

本教程介绍了简单而有趣的“Kaleidoscope”语言，通过多章的迭代式构建，展示了如何一步步构建它。这使我们能够涵盖语言设计和LLVM相关的主题，在整个过程中展示和解释代码，并减少大量的前期细节。强烈建议你亲自尝试这些代码一一复制，修改并尝试运行一遍。

警告：为了专注于教授编译器技术和LLVM，本教程不涉及软件工程的最佳实践。例如，代码使用了全局变量，不使用访问者模式等等，而是保持简单，专注于所涉及的主题。

本教程分成若干章节，每一章节都涉及单独的主题，允许你随时跳过。

  • 第1章：Kaleidoscope语言和Lexer - 我们将从这里开始，并构建我们想要构建的基本功能。Lexer是构建语言解析器的第一步，在这里我们使用一个简单的C++ Lexer以便理解。

  • 第2章：实现Parser和AST - 我们已经有了Lexer，可以开始讨论解析技术和基本的AST构造。本教程介绍了递归下降解析和运算符优先级解析。

  • 第3章：代码生成到LLVM IR - 我们已经有了AST，可以很容易地生成LLVM IR。本教程展示了如何将LLVM集成到你的项目中。

  • 第4章：添加JIT和优化支持 - LLVM有着很好的JIT支持，所以我们将深入讨论它，并展示了如何添加JIT支持。后续章节将展示如何生成.o文件。

  • 第5章：扩展语言：控制流 - 我们已经有了一个基本的语言，可以扩展它，添加控制流操作（if语句和for循环）。这给我们机会来讨论SSA构造和控制流。

  • 第6章：扩展语言：用户定义运算符 - 本章扩展了语言，允许用户定义任意的一元和二元运算符，并赋予它们的优先级。这使我们可以构建一个重要的“语言”作为库例程。

  • 第7章：扩展语言：可变变量 - 本章讨论了添加用户定义的局部变量和赋值运算符。这展示了LLVM构造SSA形式的简单性：LLVM不需要你的前端构造SSA形式，就可以使用它！

  • 第8章：编译为对象文件 - 本章解释了如何将LLVM IR编译为对象文件，就像静态编译器一样。

  • 第9章：调试信息 - 一个真正的语言需要支持调试器，所以我们添加了调试信息，允许在Kaleidoscope函数中设置断点，打印参数变量，调用函数！

  • 第10章：结论和其他花絮 - 这一章通过讨论扩展语言的方法来总结本系列，并包含了一些关于“特殊主题”的信息的指导，添加垃圾回收支持，异常处理，调试支持，“意大利面条式堆栈”支持等等。

在本教程结束时，我们将编写不到1000行（无注释、无空白）的代码行。有了这一小部分代码，我们就为一种重要的语言构建了一个漂亮的小编译器，包括一个手写的词法分析器、解析器、AST以及代码生成支持——包括静态和JIT！这充分证明了LLVM的优势，也说明了为什么对于语言设计师和其他需要高性能代码生成的人来说，LLVM是一个非常受欢迎的目标。

godot复刻游戏玩法.00.贪吃蛇

发表于 2025-05-07 分类于 计算机技术 ， 游戏开发 ， 原型设计

在本文中将以游戏引擎godot 4.4作为演示，逐步复刻出经典的贪吃蛇游戏玩法。

规则

贪吃蛇，一个经典不衰的游戏玩法，在一个有限大小的房间中你需要扮演一条永远都吃不饱的蛇在屏幕中找寻并吞吃随机出现在各处的食物。

作为一条蛇，你每吃一口食物都会让你的体型成长一点，而在有限大小的空间中这意味着食物所可能出现的随机位置更少，同时你的可行动空间也会变小。随着体型越来越长，玩家必须尝试合理的安排蛇身的空间布局，这样才能诱引随机的食物出现在自己可以轻松抵达的区域内。

准备

工程

打开godot项目管理器，创建一个名为snake的新项目。

打开项目设置，调整需要的参数：

  • 显示/窗口/拉伸/模式 -> viewport
  • 显示/窗口/拉伸/比例 -> keep_height
  • 显示/窗口/拉伸/缩放模式 -> integer
  • 渲染/纹理/画布纹理/默认纹理过滤 -> Nearest
  • 渲染/环境/默认清屏颜色 -> 纯黑色(0, 0, 0, 255)

  • 编辑器/命名/节点名称编号分隔符 -> Underscore
  • 编辑器/命名/节点名称大小写 -> snake_case
  • 编辑器/命名/脚本名称大小写 -> snake_case
