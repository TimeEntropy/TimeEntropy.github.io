godot复刻游戏玩法.00.贪吃蛇

发表于 2025-05-04 更新于 2025-05-06 分类于 计算机技术 ， 游戏开发 ， 原型设计

在本文中将以游戏引擎godot 4.4作为演示，逐步复刻出经典的贪吃蛇游戏玩法。

规则

贪吃蛇，一个经典不衰的游戏玩法，在一个有限大小的房间中你需要扮演一条永远都吃不饱的蛇在屏幕中找寻并吞吃随机出现在各处的食物。

作为一条蛇，你每吃一口食物都会让你的体型成长一点，而在有限大小的空间中这意味着食物所可能出现的随机位置更少，同时你的可行动空间也会变小。随着体型越来越长，玩家必须尝试合理的安排蛇身的空间布局，这样才能诱引随机的食物出现在自己可以轻松抵达的区域内。

准备

工程

打开godot项目管理器，创建一个名为snake的新项目。

打开项目设置，调整需要的参数：

  • 显示/窗口/拉伸/模式 -> viewport
  • 显示/窗口/拉伸/比例 -> keep_height
  • 显示/窗口/拉伸/缩放模式 -> integer
  • 渲染/纹理/画布纹理/默认纹理过滤 -> Nearest
  • 渲染/环境/默认清屏颜色 -> 纯黑色(0, 0, 0, 255)

我个人还会喜欢设置：

  • 编辑器/命名/节点名称编号分隔符 -> Underscore
  • 编辑器/命名/节点名称大小写 -> snake_case
  • 编辑器/命名/脚本名称大小写 -> snake_case

哈哈，只是个人喜欢的命名习惯而已……😓

设计

首先得有一个容纳所有必要元素的场景。

在scenes目录下创建game2D场景，就在这里设计并测试我们的游戏实体吧……

房间

game节点下创建如下节点结构：

  • 设置border节点属性Mouse/filter -> Ignore

场景面板中右键room节点，选择将分支保存为场景，保存至res://prefabs/room.tscn

打开预制体room，在根节点添加脚本res://prefabs/room.gd，添加内容如下：

  • init函数根据room_size对网格中可见的Sprite2D进行排布
  • coords_to_position函数协助外部节点定位房间内的坐标

为了让border节点显示成为一个简单的方框，我们需要在ui目录下创建一个Theme资源——取名为ui_default_theme.tres。

打开项目设置，修改选项GUI/主题/自定义，选择我们刚刚创建的res://ui/ui_default_theme.tres文件，保存并重启该项目。

点击res://ui/ui_default_theme.tres文件以打开主题编辑面板：

  • 添加修改类型PanelContainer
  • 选择覆盖名为panel的StyleBox样式
  • 创建StyleBoxFlat样式赋予
  • 修改属性：
    • bg_color -> 纯黑色(0, 0, 0, 255)
    • border_width_left -> 1
    • border_width_top -> 1
    • border_width_right -> 1
    • border_width_bottom -> 1
    • border_color -> 纯白色(255, 255, 255, 255)

场景面板中右键snake节点，选择将分支保存为场景，保存至res://prefabs/snake.tscn

打开预制体snake，在根节点添加脚本res://prefabs/snake.gd，添加内容如下：

  • _update_view函数管理grid下生成的cell复制体，可视的cells与bodies存储的蛇身坐标一致（经由room的coords_to_position函数调整）。
  • init函数用于初始化蛇身的坐标、长度以及初始前进方向。
  • move函数用于让蛇朝指定方向走一格，并且检测是否会吃到自己以及撞墙，死亡时触发die信号。
  • grow函数用于让蛇朝指定方向成长一格。

回到game场景，添加Camera2D节点，将其Camera2D/zoom属性设置为(8, 8)。

设置snake的room导出变量为room节点，按下F6测试game场景：

  • _update_view函数直接更新cell节点的坐标，因为房间内同时只能存在一个食物，所以无需考虑管理更多的cell复制节点。
