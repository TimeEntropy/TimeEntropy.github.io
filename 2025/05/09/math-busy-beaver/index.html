数学游戏：忙碌海狸（Busy Beaver）

发表于 2025-05-09 分类于 数学

规则

对于每个图灵机都会包含三个组件：

  • 一条无限长的磁带：以单元格形式存储数据，数据为0或1。
  • 一个读写头：用于读写数据并控制磁带左右移动。
  • 一组指令集：2列N行的程序表格（如下）第一列描述在磁带上读到0时执行的指令，第二列描述在磁带上读到1时执行的指令；有一个特殊的指令HALT，该指令可以设置在任意程序集单元格中；除HALT停机指令外，其余都为描述了写入数字、读写头移动方向、跳转规则三个信息的指令，例如1RB则表示：“在磁带上写入1，读写头向右移动，跳转到规则B”。
HEAD 0 HEAD 1
RULE A 1RB 0LC
RULE B 1LA 0RC
RULE C 1RA HALT

游戏让一组各自拥有不同规则的图灵机一起启动，每台图灵机初始都会将磁带所有数据设为0，初始规则设为RULE A。

在经历一连串指令操作后会出现有图灵机无限执行指令无法停下的趋势，也会有图灵机运行一段时间后突然停下的事件发生

游戏目标就是要找出组中最后停机的那台图灵机，并推测（统计）他停机前执行了多少步指令——同时这台图灵机也会获得“忙碌海狸（Busy Beaver）”的称号！

组内图灵机的数量其实由规则的数量N决定：规则集中的每个单元格都有可能出现4N + 1种情况的指令，共计2N个单元格，N条规则时组内图灵机的数量为(4N + 1)2N。
